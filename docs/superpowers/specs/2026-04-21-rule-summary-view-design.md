# 规则汇总界面设计文档

**日期**: 2026-04-21
**主题**: 规则面板字段汇总视图
**状态**: 已确认

## 背景与问题

在 Legado 书源生成器的「规则」模式下，每个规则类型（搜索页、发现页、详情页、目录页、正文页）包含多个字段。用户目前在字段间通过「上一步 / 下一步」线性导航，虽然已支持字段状态摘要的点击跳转，但仍然缺乏一个全局视角来一次性查看和修改当前规则类型的所有字段配置。

## 目标

为每个规则类型增加一个「汇总」虚拟步骤，以列表形式展示该规则类型下所有字段的配置状态、规则值，并支持 inline 编辑和快速跳转到对应字段。

## 方案选择

经过对比，采用 **方案 A+：嵌入字段流程末尾作为虚拟"汇总"步骤，结合字段状态摘要点击导航实现随时访问**。

- **方案 A**：将"汇总"作为每个规则类型的虚拟最后一步，用户通过「下一步」走到最后时自然进入。结合已有的字段状态摘要点击导航，用户可随时点击"汇总"项直接跳转。
- **方案 B**：新增顶层 Mode Tab。优点是随时可访问，缺点是 tab 数量过多、功能边界模糊。
- **方案 C**：在"规则"Tab 内增加子切换。优点是不增加顶层 tab，缺点是多了一层内部导航，与用户习惯不符。

## 设计细节

### 交互逻辑

1. 每个规则类型的字段列表末尾虚拟追加一个"汇总"项，索引为 `fields.length`。
2. **字段状态摘要**中显示所有真实字段 + 一个"汇总"项（图标为 ☰ 或 ≡）。
3. **通过「下一步」访问**：用户走到规则类型的最后一个真实字段后，再点「下一步」进入汇总页，此时按钮显示「完成」。
4. **通过点击摘要访问**：用户可随时点击字段状态摘要中的"汇总"项，直接跳转到汇总页。
5. **从汇总页跳转回字段**：汇总页中每行字段名可点击，点击后调用 `goToStep(index)` 跳转到对应字段。
6. **inline 编辑**：汇总页中每行的规则值使用 textarea，支持直接修改，onblur 时自动同步到 `state.rules[activeRuleType].fields[key].value`，并触发 `saveState()` 和 `renderFieldStatusSummary()`。

### 数据模型变更

- `getFields()` 返回的数组在渲染时不改变，但在 `renderFields()` 和 `updateStepIndicator()` 中需要识别"汇总"步骤的特殊处理。
- 汇总步骤不是一个真实字段，没有 `f.key`，因此在 `renderFields()` 中需要判断 `rule.currentStep === fields.length` 时渲染汇总视图而非普通字段视图。
- `handleNext()` 中，当 `rule.currentStep === fields.length - 1`（最后一个真实字段）时，点击「下一步」应进入汇总步骤（`rule.currentStep++` 变为 `fields.length`）。
- 当处于汇总步骤时，「下一步」按钮显示「完成」，点击后触发 `handleExport()`。

### 视觉设计

- **汇总容器**：使用与 `.field-item` 类似的卡片风格（白色背景、圆角、边框）。
- **列表行**：每行一个字段，flex 布局，左侧为字段名（带状态图标），右侧为规则值 textarea。
- **字段名样式**：可点击跳转，hover 时显示下划线和主题色，cursor: pointer。
- **规则值 textarea**：使用 `.input` 样式，只读时显示为普通文本，点击后变为可编辑。
- **当前汇总项高亮**：字段状态摘要中的"汇总"项在激活时与其他 active 字段一致（accent-dim 背景 + 加粗）。
- **过渡**：hover 和状态变化保持 0.15s 过渡。

### 可访问性

- 可点击的字段名和"汇总"项均提供 hover 反馈。
- inline 编辑的 textarea 在获得焦点时有清晰的 focus ring。
- 不依赖颜色作为唯一信息来源：状态使用图标 + 文字双重标识。

### 边界情况

- **无字段配置**：汇总页正常显示，所有字段的规则值为空。
- **切换 ruleType**：汇总页内容自动刷新为当前规则类型的字段。
- **处于汇总页时切换 mode**：正常隐藏，切回"规则"模式后恢复之前的步骤状态。
- **inline 编辑为空值**：允许清空，与现有字段逻辑一致。
- **手动输入状态**：汇总页显示的规则值与字段编辑页一致（已处理 webView 后缀）。

## 实现范围

- `src/popup/popup.js`：
  - 修改 `updateStepIndicator()`：识别汇总步骤并显示"汇总"标题。
  - 修改 `renderFields()`：当 `rule.currentStep === fields.length` 时渲染汇总视图。
  - 新增 `renderSummaryView()`：渲染汇总列表。
  - 修改 `handleNext()`：处理从最后一个真实字段进入汇总步骤，以及从汇总步骤触发导出。
  - 修改 `updateNavButtons()`：汇总步骤时「下一步」显示「完成」。
  - 修改 `renderFieldStatusSummary()`：添加"汇总"虚拟项。
- `src/popup/popup.css`：添加汇总视图相关样式。
- `src-firefox/popup/popup.js`：同步 JS 改动。
- `src-firefox/popup/popup.css`：同步 CSS 改动。

## 不做的范围（YAGNI）

- 不新增独立的 HTML 容器，复用现有的 `#fieldContainer`。
- 不新增顶层 Mode Tab。
- 不增加字段的删除/重排功能（超出当前需求）。
- 不增加跨规则类型的全局汇总（只汇总当前规则类型）。
- 不增加规则值的实时预览/验证。

## 验收标准

- [ ] 每个规则类型增加虚拟"汇总"步骤，可通过「下一步」进入。
- [ ] 字段状态摘要显示"汇总"项，点击可跳转。
- [ ] 汇总页以列表形式展示当前规则类型的所有字段、状态图标、规则值。
- [ ] 点击汇总页中的字段名可跳转到对应字段。
- [ ] 汇总页中的规则值支持 inline 编辑，修改后自动保存。
- [ ] 汇总步骤时「下一步」按钮显示「完成」，点击后触发导出。
